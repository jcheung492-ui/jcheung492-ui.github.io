# 域名接入操作手册 · `sound-truth.com` → GitHub Pages

> 本文件是本地工作文档，已加入 `.gitignore`，**不会上线**。
> 目标：把 Cloudflare 上的域名 `sound-truth.com` 接到现有的 GitHub Pages 站点，全程**免费**、**不改任何网站代码**。

---

## 0. 背景与原则

**当前架构**
- 站点：GitHub Pages，仓库 `jcheung492-ui/jcheung492-ui.github.io`，`master` 分支＝线上，现址 https://jcheung492-ui.github.io/
- 媒体：音频/视频在 Cloudflare R2（`MEDIA_BASE = https://pub-22e48f3da3c74cabbfdd79aec53da6ad.r2.dev`），封面/图廊仍在仓库
- 域名：`sound-truth.com`，在 Cloudflare（注册商 + DNS，当前已是 Full/橙云代理）
- 发布链路：管理面板「发布到线上」走 GitHub API 提交到仓库 —— **与访问域名无关，接域名后照常工作**

**两条铁律**
1. **先灰云、后橙云**：GitHub 要用 Let's Encrypt 给 `sound-truth.com` 签 HTTPS 证书，Cloudflare 橙色代理会挡住验证。所以先用「DNS only / 灰云」让 GitHub 签好证书、站点能开，再决定要不要开橙云加速。
2. **橙云时 SSL 必须设 `Full`**，绝不能用 `Flexible`（会和 GitHub 强制 HTTPS 打架，跳转死循环）。

**钱**：域名注册费已付（约 $10/年续费）。DNS 解析、橙云代理/CDN、HTTPS 证书全部 **¥0**。Pro 套餐（$20/月起）那些高级功能本站用不到。

**关键常量（照抄用）**
- GitHub Pages 官方 4 个 IP：`185.199.108.153` / `185.199.109.153` / `185.199.110.153` / `185.199.111.153`
- `www` 的 CNAME 目标：`jcheung492-ui.github.io`

## 步骤 1 · Cloudflare 加 DNS 记录（灰云）

**操作**
1. 进 Cloudflare → 选 `sound-truth.com` → 左侧 **DNS → Records**。
2. **先删除**注册时自动生成的占位记录：任何指向 `@`（根域）或 `www` 的 A / AAAA / CNAME（常见是假地址 `192.0.2.1`）。
3. 新增以下 5 条记录，**代理状态全部选「DNS only / 灰云」**：

   | 类型 | 名称 | 内容 | 代理 |
   |------|------|------|------|
   | A | `@` | `185.199.108.153` | 灰云 |
   | A | `@` | `185.199.109.153` | 灰云 |
   | A | `@` | `185.199.110.153` | 灰云 |
   | A | `@` | `185.199.111.153` | 灰云 |
   | CNAME | `www` | `jcheung492-ui.github.io` | 灰云 |

**✅ 验收**
- DNS Records 列表里能看到这 5 条，且 4 条 A 记录的代理列显示灰色云朵（"DNS only"），没有多余的 `@`/`www` 旧记录。
- 命令行（等几分钟 DNS 生效后）：
  ```bash
  dig +short sound-truth.com
  # 期望输出这 4 个：185.199.108.153 / 109.153 / 110.153 / 111.153
  dig +short www.sound-truth.com
  # 期望输出 jcheung492-ui.github.io 以及上面那组 IP
  ```

**卡住怎么办**
- `dig` 没结果或还是旧值：DNS 有缓存，等 5–30 分钟再试；确认记录确实保存了。
- 出现 `192.0.2.x` 之类：占位记录没删干净，回去删掉。

## 步骤 2 · GitHub 设自定义域名 + 等签证书

**操作**
1. 打开仓库 `jcheung492-ui/jcheung492-ui.github.io` → **Settings → Pages**。
2. **Custom domain** 填 `sound-truth.com` → **Save**。
   - GitHub 会自动往仓库提交一个 `CNAME` 文件（内容就是 `sound-truth.com`）。**别手动删它**；本地下次 `git pull` 会拉到。
3. GitHub 先做 DNS 校验（通过后域名旁出现绿勾），随后自动申请 HTTPS 证书。
   - **耐心**：从几分钟到 ~1 小时不等，期间可能显示 "Certificate not yet created" 或类似提示，正常。

**✅ 验收**
- Settings → Pages 里 Custom domain 显示 `sound-truth.com` 且带**绿勾**（DNS check successful）。
- 页面出现 **"Your site is published at https://sound-truth.com"**。
- 仓库根目录多了一个 `CNAME` 文件（GitHub 自动提交的）。

**卡住怎么办**
- 一直 "DNS check in progress"：多半是步骤 1 的 DNS 还没生效，等一会儿；或 `dig` 验证 IP 是否已正确指向那 4 个地址。
- 报 "domain already taken"：该域名在别的 GitHub 账号配过；本场景全新域名，不会遇到。

## 步骤 3 · 验证 HTTPS 能开站

**操作 / ✅ 验收**
1. 浏览器开 `https://sound-truth.com`：能看到你的站，地址栏是绿锁（有效证书）。
2. 浏览器开 `https://www.sound-truth.com`：能打开（GitHub 会把 www 跳到根域，或反之，都算正常）。
3. 命令行复核：
   ```bash
   curl -sI https://sound-truth.com | head -n 1
   # 期望：HTTP/2 200

   echo | openssl s_client -servername sound-truth.com -connect sound-truth.com:443 2>/dev/null \
     | openssl x509 -noout -subject -issuer
   # 期望 subject 含 sound-truth.com；issuer 是 Let's Encrypt（灰云阶段）
   ```
4. 点几下站内导航 / 播放一首歌 / 看封面：页面、音频（R2）、封面都正常加载（确认换域名没影响相对路径与 R2）。

**卡住怎么办**
- 证书错误 / 不安全：GitHub 证书还没签好，回步骤 2 再等等。
- 站能开但**音频放不出**：检查浏览器控制台有无对 `pub-...r2.dev` 的报错；R2 是独立绝对地址，理论上不受域名影响。

## 步骤 4 · 勾选 Enforce HTTPS

**操作**
- 步骤 3 通过、证书就绪后，回 GitHub **Settings → Pages**，勾上 **Enforce HTTPS**。

**✅ 验收**
- 访问 `http://sound-truth.com`（明文 http）会自动 301 跳到 `https://`。
  ```bash
  curl -sI http://sound-truth.com | grep -i location
  # 期望：location: https://sound-truth.com/
  ```

**到这里，灰云方案已全部完成，域名正式可用。下面是可选项。**

> ### 📌 换域名后的管理面板（必读一次）
> 浏览器按「同源」隔离本地存储，新域名 = 全新一套，旧地址的东西不会自动搬过来：
> - **解锁入口**：在新域名访问一次 `https://sound-truth.com/#studio`，页脚就出现「管理」按钮（记在本机）。以后日常只点页脚「管理」。
> - **重填一次**：进面板后「设置 Token」重粘 GitHub Token、「设置上传密码」重填 R2 上传密码（`bx-gh-token` / `bx-upload-token` 按域名隔离，不会从 github.io 带过来）。
> - **旧地址作废**：设了自定义域名后 `jcheung492-ui.github.io` 会 301 跳到 `sound-truth.com`，不再是独立入口。已发布内容都在仓库、照常显示；旧域名上若有「未发布的本地草稿」则跳转后够不着。

## 步骤 5 ·（可选）开 Cloudflare 橙云加速

> 想要 CDN 缓存/更快/基础防护再做；不做也完全没问题。**免费**。

**操作**
1. Cloudflare → DNS → Records：把步骤 1 那 5 条记录的代理状态从**灰云改成橙云（Proxied）**。
2. Cloudflare → **SSL/TLS → Overview**：加密模式设为 **`Full`**。
   - ⚠️ **绝不要选 `Flexible`** —— 会与 GitHub 强制 HTTPS 冲突，造成无限跳转。
   - `Full (strict)` 也可（GitHub 证书有效时），但 `Full` 最稳，先用 `Full`。

**✅ 验收**
- 站点仍能正常开 `https://sound-truth.com`，无跳转死循环（浏览器不报 "重定向次数过多"）。
- 响应头出现 Cloudflare 特征：
  ```bash
  curl -sI https://sound-truth.com | grep -iE 'server|cf-ray'
  # 期望：server: cloudflare，且有一行 cf-ray: ...
  ```
- 再点几下站、放首歌，一切正常。

**卡住怎么办（出现跳转死循环 / ERR_TOO_MANY_REDIRECTS）**
- 99% 是 SSL 模式选成了 `Flexible`，改回 `Full` 即可。
- 仍异常：把记录切回**灰云**，站立刻恢复（灰云方案永远是安全回退点）。

## 步骤 6 ·（更以后·可选）媒体绑自己的子域名

> 现在 r2.dev 工作正常，可不动。将来想去掉那串 `pub-...r2.dev`、统一到自己域名时再做。

**操作**
1. Cloudflare → **R2 → 桶 `justin-media` → Settings → Custom Domains** → 添加 `media.sound-truth.com`（Cloudflare 会自动加 DNS + 签证书）。
2. 改 `js/media.js`：`MEDIA_BASE` 从 `https://pub-22e48f3da3c74cabbfdd79aec53da6ad.r2.dev` 换成 `https://media.sound-truth.com`（结尾不带斜杠）。
3. 资源缓存版本 `?v=N` 整体 +1，本地硬刷验证音频能放，再发布上线。

**✅ 验收**
- `https://media.sound-truth.com/audio/<某文件>.mp3` 能直接在浏览器播放。
- 站内播放正常，控制台无 404/跨域报错。

**回退**：把 `MEDIA_BASE` 改回原 r2.dev 地址即可。
