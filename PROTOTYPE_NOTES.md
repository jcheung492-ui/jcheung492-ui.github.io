# Prototype Notes

## Question

Can the current personal site become more clearly "music / studio / portfolio" without changing the user's actual biography, photos, contacts, or project claims?

## Answer

Yes. The safest first pass is a visual-system prototype:

- tighter typography,
- restrained waveform motif,
- stronger project meta styling,
- static-site-safe Studio theme toggle,
- subtle playback equalizer,
- mobile nav tightening.

## Deliberately Not Changed

- No fake client names, credits, city, email, social links, or project history.
- No replacement of portrait, candid photo, gallery, covers, or audio assets.
- No backend migration for guestbook/comments/library.
- No new external font dependency.

## Next Decision

If this visual direction is accepted, the next concrete step should be either:

- provide real photos, contact links, and platform links for content replacement; or
- continue refining interaction/layout in this prototype before folding selected changes back into the final single-file version.
